# 淀粉贸易业务管理系统 · AI Agent 驱动的进销存+财务一体化

> 用飞书多维表格替代传统ERP，用自然语言对话替代手工录入。三个AI Agent（DeepSeek + 扣子 + Hanako）协作实现采购、销售、物流、财务的全链路自动化。

---

## 1. 一句话定位

这是一个**用飞书多维表格 + AI Agent 替代传统ERP系统**的实践方案。13张表、60+公式字段、200+条业务记录，通过自然语言对话即可完成采购、销售、回款、货运、财务的全部操作，无需打开表格、无需写SQL、无需手工对账。

---

## 2. 解决什么痛点

**以前是这样的**：
- 打开飞书表格，逐个单元格填写客户名、商品、吨数、单价、关联订单号
- 回款到账后手动找对应订单，计算待回金额，手写备注
- 同一条货运记录在"港口装货"和"到货处理"里分别填两次
- 回款超额了不知道怎么分，100,000元挂在一个30,000的订单上
- 公式字段换了表格就丢失，60+公式要重写

**现在是这样的**：
- 说一句话，AI自动解析分类、创建记录、匹配关联、验证完整性
- 回款自动FIFO分配到最早未结清的订单，超额自动拆为多条
- 货运事件2自动识别同车牌号，更新已有记录而非新建
- 全量检查：回款率不超100%、关联不跨客户、发货不超量
- 公式读取`formula_expression`反向导出，断链可追溯

**适合谁用**：
- 中小贸易商：用飞书替代传统进销存软件，零代码、零运维
- 个人经营者：业务+个人财务在一张表里管完
- AI开发者：研究AI Agent在真实业务场景落地的完整案例

## 3. 核心功能

| 功能 | 解决什么问题 |
|------|------------|
| 自然语言批量录入 | 发送`【销售订单：王奎 订货32吨绿五叶...】`，自动创建+关联+验证 |
| DuplexLink双向写入 | 避免飞书API的"空壳链接"陷阱，code=0后再回读验证 |
| FIFO回款拆分 | 一笔回款自动按订单时间顺序分配，超额拆为多条，不跨客户 |
| 货运到货联动 | 事件2（到货处理）自动识别同车牌号，UPDATE已有记录而非新建 |
| 12条数据铁律 | 覆盖浮点精度、空数组陷阱、反向链接覆盖等全部已知坑点 |
| 全量完成检查 | 客户无重复、回款率≤100%、发货率≤100%、关联不跨客户 |
| 微信提现自动化 | 3步自动流程：工作支出(手续费)+工作支出(到账)+回款明细(无客户) |
| 多Base兼容 | 主Base和Wiki副本两套表ID自动切换，账户名差异自动适配 |

## 4. 使用方法（场景化）

**场景一：日常业务录入**

*什么时候用：收到供应商发货、客户下订单、客户回款时*

1. 用自然语言描述业务：`【回款明细：【王奎👥】回款到【微信钱包💳】607,000元💰...】`
2. Agent自动解析分类 → 预查档案 → 创建记录 → 匹配关联 → 回读验证
3. 无超标、无跨客户错配 → 报告结果

**场景二：回款超额处理**

*什么时候用：客户回款金额>单一订单金额时*

1. 发送回款提示词，Agent检测到回款率将超过100%
2. 自动FIFO拆分：按订单时间排序 → 逐单分配 → 到100%后顺延
3. 删旧建新：从1条回款拆为2-3条，每条只关联一个订单

**场景三：微信提现**

*什么时候用：从微信钱包转钱到银行卡时*

1. 发送：`【账户转账：从【微信钱包💳】提现 50,000元💰到【农业银行🏦】，手续费 0.1%】`
2. Agent自动计算：手续费=50，到账=49,950
3. 创建3条记录：工作支出(50)+工作支出(49,950)+回款明细(49,950，无客户关联)

## 7. 常见问题

**Q: 为什么回款金额全是0？**
A: 公式字段从原Base复制到Wiki副本时丢失了。需要读取原Base的`formula_expression`，然后在UI里手动填入。飞书API不支持程序写入公式。

**Q: 回款率为什么超过100%？**
A: 一条回款关联了多个订单，SUM公式把全额加到了每个订单上。解决：FIFO拆分，一条回款只绑一个订单。

**Q: 送货关联订单后`对应订单`字段为空？**
A: DuplexLink格式错误。必须用纯数组`["record_id"]`，不能用对象格式`{table_id:..., record_ids:...}`。code=0不等于链接生效，必须回读验证。

**Q: 浏览器操作经常失败？**
A: 飞书API首页有CSP限制，需要navigate到`/open-apis/auth/v3/tenant_access_token/internal`（无CSP）再执行。多出口IP需要全部加白名单。OAuth用户token不受IP限制。

**Q: 反向链接为什么消失了？**
A: PUT写反向链接时覆盖了旧值。正确做法：先GET现有ids → 拼接`[...old, new]` → PUT。
